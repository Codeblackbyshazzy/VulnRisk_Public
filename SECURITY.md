# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

We take security seriously. If you discover a security vulnerability, please follow these steps:

### **1. Create a private security issue**
Create a new issue with the "security" label at: https://github.com/GurkhaShieldForce/vulnrisk-public/issues/new

### **2. Include the following information**
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)
- Your contact information

### **3. Response timeline**
- **Acknowledgment**: Within 24 hours
- **Initial assessment**: Within 72 hours
- **Resolution**: Within 30 days (depending on severity)

## Security Best Practices

### **For Contributors**
- Never commit secrets or credentials
- Use environment variables for sensitive data
- Follow secure coding practices
- Report security issues responsibly

### **For Users**
- Keep your installation updated
- Use strong API keys
- Enable authentication
- Monitor for security updates

## Security Features

VulnRisk includes several security features:

- **Input Validation**: Protection against injection attacks
- **Authentication**: JWT-based authentication
- **API Security**: Rate limiting and API key management
- **Data Protection**: Encryption at rest and in transit
- **Audit Logging**: Comprehensive activity tracking

## Security Updates

Security updates are released as needed. We recommend:

- Subscribing to security notifications
- Following our GitHub releases
- Monitoring our security advisory page
